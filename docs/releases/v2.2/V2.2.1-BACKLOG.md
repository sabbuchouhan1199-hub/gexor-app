# Gexor v2.2.1 Backlog & Future Vision

This document tracks items explicitly deferred from the Gexor v2.2.0 freeze.

---

## 1. Deferred Patch-Level Work (Target: v2.2.1)

- **`V22-MAINT-01`: Automatic Event Replay Log Compaction**:
  - *Description*: Implementation of background compaction and pruning routines for historical `execution_events` sequences after executions reach terminal states (`completed`, `failed`, `cancelled`, `dead_letter`).
  - *Scope*: Single-node SQLite event retention policy (e.g. prune delta events after snapshot retention threshold).
  - *Impact*: Prevents SQLite database growth over extended runtime operation.

---

## 2. Strategic Future Architecture (Target: v2.3+ / Future Major Releases)

- **`V22-DIST-01`: Multi-Node Redis Queue & High Availability**:
  - *Description*: Transitioning execution job queueing from single-node SQLite leases to distributed Redis streams/queues and pub/sub event broadcasting.
  - *Scope*: Distributed API rate-limiting store, multi-node worker clustering, and high-availability database replication.

- **Long-Term Memory & Vector RAG Pipeline**:
  - *Description*: Automatic memory extraction from conversation histories, dense vector embedding generation, and semantic RAG retrieval over user documents.
  - *Scope*: Vector database integration, embedding provider adapters, and semantic search.
